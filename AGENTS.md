The canonical Phase B handoff is therefore: 1. **Builder implementation finishes**. Do NOT call `kanban_complete()` (that would set the card to `done` and lose the audit handoff). Promotion requires a manual or Prime-side DB write because no worker-callable move-to-`review` tool exists. The required steps are: - `UPDATE tasks SET status='review', assignee='auditor', claim_lock=NULL, claim_expires=NULL WHERE id = <card>` (single transaction); - append a `transitioned` event recording that the card moved from `blocked` to `review` and from `assignee='builder'` to `assignee='auditor'`; - record a `kanban_comment()` on the same card with the Builder's `review-required` reason and the full review packet so the Auditor inherits the Builder's raw evidence in the same card context. - When Prime cannot perform the DB write (e.g. constrained to the `kanban_*` tool surface and the active Hermes build lacks `kanban_set_status`), the **operator** performs the same transaction as a one-shot DB write.
